# v0.2.0 Implementation Plan

Derived from [Apple Design Award criteria](apple-design-award-criteria.md) and [macOS App Design Rules](macos-app-design-rules.md). Each item references the specific rule it addresses.

---

## Current State (v0.1.x) — What's Wrong

| Problem | Design Rule Violated |
|---|---|
| Uses `main.swift` + `NSApplicationDelegate` instead of SwiftUI `@main App` | Rules 1.6, 1.7, 10 — no standard menu bar, no `Settings` scene |
| Settings uses `GroupBox` in a manual `NSWindow` | Rule 1.7 — must use `TabView` + `.formStyle(.grouped)` in a `Settings` scene |
| Menu bar icon doesn't set `isTemplate = true` | Rule 3.2 — icon breaks in dark mode and tinted menu bars |
| No standard app menu bar (About, Edit, Window, Help) | Rule 1.6 — all are mandatory |
| No keyboard shortcuts except Save/Cancel in settings | Rule 1.8 — Cmd+Q, Cmd+W, Cmd+, must all work |
| No VoiceOver labels on any element | Rule 5.1 — every interactive element must be labelled |
| No right-click on status item | Section 7 menu bar patterns — expected by all users |
| Uses `.foregroundColor(.red)` hardcoded | Rule 4.1 — must use semantic colours |
| Fixed-size settings window (520x460 hardcoded) | Rule 1.3 — settings can be fixed, but size should fit content |
| Targets macOS 13 — no `@Observable` | Rule 2.4 — target macOS 14 for modern APIs |
| No notarization — Gatekeeper blocks app | Rule 7.2 — notarization is mandatory for direct distribution |
| No auto-update mechanism | Rule 7.4 — Sparkle is the standard |
| Config stored as plaintext TOML, no Keychain | Rule 6.2 — acceptable for config, but noted |

## Architecture Overhaul: SwiftUI `@main` App

The single biggest change. Replaces the manual AppKit setup with the standard SwiftUI app lifecycle.

### Why

- Automatic `Settings` scene with Cmd+, shortcut
- Automatic standard menu bar (App, Edit, Window, Help)
- `.commands {}` modifier for custom menu commands
- Window state restoration for free
- Clean separation of concerns

### File Changes

**Delete:** `main.swift` (incompatible with `@main`)

**New:** `MenuBarApp.swift`
```swift
import SwiftUI
import HydrationCore

@main
struct MenuBarApp: App {
    @State private var appState = AppState()

    var body: some Scene {
        MenuBarExtra {
            StatusPanelView(state: appState)
        } label: {
            Image(systemName: appState.statusIcon)
                .accessibilityLabel(appState.statusDescription)
        }

        Settings {
            SettingsView()
                .environmentObject(appState)
        }

        // Hidden window for About panel
        Window("About iCloud NFS Exporter", id: "about") {
            AboutView()
        }
        .defaultSize(width: 300, height: 200)
        .windowResizability(.contentSize)
    }
}
```

**New:** `AppState.swift` — replaces AppDelegate's state management
```swift
import SwiftUI
import HydrationCore

@Observable
final class AppState {
    var config = ConfigReader.load()
    var daemonRunning = false
    private var timer: Timer?

    var statusIcon: String {
        daemonRunning ? "cloud.fill" : "cloud"
    }

    var statusDescription: String {
        "iCloud NFS Exporter — daemon \(daemonRunning ? "running" : "stopped")"
    }

    init() { startPolling() }

    func refresh() {
        config = ConfigReader.load()
        checkStatus()
    }

    // ...
}
```

**Renamed:** `AppDelegate.swift` → deleted (logic moves to `AppState`)

### Package.swift

Bump minimum deployment:
```swift
platforms: [.macOS(.v14)],
```

### Impact
- `@Observable` requires macOS 14 (Sonoma, released Oct 2023 — 2.5 years old)
- `MenuBarExtra` requires macOS 13 — already met
- `Settings` scene is automatic — no manual window management

## Settings Window: TabView + Grouped Form

### Current
Manual `GroupBox` in an `NSWindow` opened via `NSHostingView`.

### Target
SwiftUI `Settings` scene with `TabView`:

```swift
struct SettingsView: View {
    var body: some View {
        TabView {
            GeneralSettingsTab()
                .tabItem { Label("General", systemImage: "gear") }
            FoldersSettingsTab()
                .tabItem { Label("Folders", systemImage: "folder") }
            NetworkSettingsTab()
                .tabItem { Label("Network", systemImage: "network") }
            AdvancedSettingsTab()
                .tabItem { Label("Advanced", systemImage: "gearshape.2") }
        }
        .frame(width: 500)
    }
}
```

Each tab uses `.formStyle(.grouped)`:

```swift
struct FoldersSettingsTab: View {
    @EnvironmentObject var appState: AppState

    var body: some View {
        Form {
            Section("Exported Folders") {
                ForEach(appState.config.folders, id: \.source) { folder in
                    LabeledContent(folder.label) {
                        Text(folder.source)
                            .foregroundStyle(.secondary)
                            .font(.caption)
                    }
                }
                .onDelete { appState.removeFolders(at: $0) }

                Menu {
                    // iCloud container picker
                } label: {
                    Label("Add Folder", systemImage: "plus")
                }
            }
        }
        .formStyle(.grouped)
    }
}
```

### Files
- `SettingsView.swift` — tab container
- `GeneralSettingsTab.swift` — socket path, mount base, launch at login
- `FoldersSettingsTab.swift` — folder list with add/remove
- `NetworkSettingsTab.swift` — NFS CIDR, server choice
- `AdvancedSettingsTab.swift` — daemon control, log path

## Menu Bar: Rich Status Panel

### Current
Plain `NSMenu` with disabled text items.

### Target
`MenuBarExtra` with a SwiftUI view for richer display:

```swift
struct StatusPanelView: View {
    let state: AppState

    var body: some View {
        VStack(alignment: .leading, spacing: 12) {
            // Header with status
            HStack {
                Image(systemName: state.statusIcon)
                    .symbolRenderingMode(.hierarchical)
                    .foregroundStyle(state.daemonRunning ? .green : .secondary)
                Text(state.daemonRunning ? "Running" : "Stopped")
                    .font(.headline)
            }
            .accessibilityElement(children: .combine)

            Divider()

            // Folder list
            if state.config.folders.isEmpty {
                Text("No folders configured")
                    .foregroundStyle(.secondary)
            } else {
                ForEach(state.config.folders, id: \.source) { folder in
                    FolderStatusRow(folder: folder)
                }
            }

            Divider()

            // Actions
            Button("Refresh") { state.refresh() }
                .keyboardShortcut("r")

            SettingsLink {
                Text("Settings...")
            }
            .keyboardShortcut(",")

            Divider()

            Button("Quit") { NSApp.terminate(nil) }
                .keyboardShortcut("q")
        }
        .padding(12)
        .frame(width: 280)
    }
}
```

### Icon States

| State | Symbol | Meaning |
|---|---|---|
| Stopped | `cloud` | Daemon not running |
| Idle | `cloud.fill` | Running, no activity |
| Hydrating | `icloud.and.arrow.down` | File download in progress |
| Error | `exclamationmark.icloud` | Last operation failed |

All icons use `isTemplate = true` automatically via SF Symbols in `MenuBarExtra`.

## Accessibility

### VoiceOver Labels

Every interactive element needs an accessibility label. Specific additions:

```swift
// Status icon in menu bar
Image(systemName: "cloud.fill")
    .accessibilityLabel("iCloud NFS Exporter — daemon running")

// Folder row
HStack { ... }
    .accessibilityElement(children: .combine)
    .accessibilityLabel("\(folder.label), mounted at \(mountPoint)")

// Remove button
Button { ... } label: { Image(systemName: "minus.circle.fill") }
    .accessibilityLabel("Remove \(folder.label)")
    .accessibilityHint("Removes this folder from NFS exports")

// Network field
TextField("CIDR", text: $network)
    .accessibilityLabel("Allowed network")
    .accessibilityHint("CIDR notation, for example 192.168.1.0/24")
```

### Keyboard Navigation

- All buttons have `.keyboardShortcut()`
- Tab order is logical within settings
- Escape dismisses the status panel
- Cmd+, opens settings (automatic with `Settings` scene)
- Cmd+Q quits (automatic with SwiftUI app lifecycle)

## Visual Polish

### Semantic Colours

Replace all hardcoded colours:

| Current | Replace With |
|---|---|
| `.foregroundColor(.red)` | `.foregroundStyle(.red)` (system red adapts to dark mode) |
| `.foregroundColor(.secondary)` | `.foregroundStyle(.secondary)` (use Style, not Color) |
| Any hardcoded RGB | System semantic colour |

### Typography

Use semantic font styles consistently:

| Current | Replace With |
|---|---|
| `.font(.title2.bold())` | `.font(.headline)` for section headers |
| `.font(.caption)` | `.font(.caption)` (already correct) |
| `.fontWeight(.medium)` | `.font(.body)` (default weight is correct) |

### Spacing

Use 8pt grid:
- `.padding(12)` → `.padding(16)` (2 grid units)
- `.spacing: 20` → `.spacing: 24` (3 grid units for sections)
- `.spacing: 8` → `.spacing: 8` (already on grid)

## Distribution

### Code Signing and Notarization

Required for Gatekeeper approval and Homebrew cask submission.

1. Obtain Apple Developer ID certificate ($99/year)
2. Add signing to build scripts:
   ```bash
   codesign --deep --force --sign "Developer ID Application: ..." \
       "iCloud NFS Exporter.app"
   ```
3. Notarize:
   ```bash
   xcrun notarytool submit "iCloud-NFS-Exporter.dmg" \
       --keychain-profile "AC_PASSWORD" --wait
   xcrun stapler staple "iCloud-NFS-Exporter.dmg"
   ```
4. Add to CI release workflow as post-build steps

### Sparkle Auto-Updates

Add the [Sparkle framework](https://sparkle-project.org/) for in-app update checking:
- SPM dependency: `https://github.com/sparkle-project/Sparkle`
- Host appcast.xml on GitHub Pages or in the repo
- Check for updates on launch + periodically
- User-facing "Check for Updates..." in app menu

### Homebrew Cask

Convert `Formula/icloud-nfs-exporter.rb` from a source formula to a Cask (binary distribution):
```ruby
cask "icloud-nfs-exporter" do
  version "0.2.0"
  sha256 "..."
  url "https://github.com/wizz-cmd/icloud-nfs-exporter/releases/download/v#{version}/iCloud-NFS-Exporter-#{version}.dmg"
  name "iCloud NFS Exporter"
  desc "Export iCloud Drive folders via NFS"
  homepage "https://github.com/wizz-cmd/icloud-nfs-exporter"
  app "iCloud NFS Exporter.app"
end
```

Requires notarization before Homebrew will accept it (mandatory since Homebrew 5.0).

## Implementation Order

### Phase 1 — Architecture (breaks everything, do first)
1. Bump deployment target to macOS 14
2. Replace `main.swift` + `AppDelegate` with `@main` SwiftUI App
3. Move state to `@Observable AppState`
4. Use `MenuBarExtra` for status panel
5. Use `Settings` scene for settings window

### Phase 2 — Settings Rebuild
6. Split into `TabView` with 4 tabs
7. Use `.formStyle(.grouped)` in each tab
8. Add General tab (launch at login, socket path)
9. Add Advanced tab (daemon control)

### Phase 3 — Accessibility + Polish
10. Add VoiceOver labels to all elements
11. Add keyboard shortcuts to all actions
12. Replace hardcoded colours with semantic styles
13. Align spacing to 8pt grid
14. Add dynamic status icons (cloud/cloud.fill/icloud.and.arrow.down)

### Phase 4 — Distribution
15. Code signing + notarization
16. Sparkle auto-updates
17. Homebrew Cask formula
18. First-launch onboarding view
