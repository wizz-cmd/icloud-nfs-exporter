# Design Recommendations for v0.2.0

Based on [Apple Design Award criteria research](apple-design-award-criteria.md). Each recommendation is tagged with the ADA category it addresses.

---

## Priority 1 — Must-fix (current app feels broken without these)

### 1.1 Use NSMenu instead of NSWindow for status display
**[Interaction]**

The current app opens a separate NSWindow for settings, but the main status display is a flat `NSMenu` with disabled text items. Best-practice menu bar apps embed a SwiftUI view inside an `NSMenuItem` via `NSHostingView`, giving a richer at-a-glance panel without opening a window.

**Recommendation:** Replace the plain `NSMenu` dropdown with a custom SwiftUI view embedded in a single `NSMenuItem`. Show daemon status, folder list, and action buttons in a compact, visually appealing panel. Keep the Settings window as a separate Cmd+, window for configuration.

### 1.2 Template icon for menu bar
**[Visuals]**

The status bar icon must have `isTemplate = true` so macOS automatically adapts it to light mode, dark mode, and menu bar tinting. Without this, the icon looks broken in some configurations.

**Recommendation:** Ensure `statusItem.button?.image?.isTemplate = true`. Use a simple cloud silhouette that reads clearly at 18×18pt. Consider dynamic states: hollow cloud (stopped), filled cloud (running), cloud with arrow (hydrating).

### 1.3 Right-click context menu
**[Interaction]**

Native menu bar apps support right-click on the status item for a minimal context menu (Quit, Preferences). Currently there is no right-click behaviour.

**Recommendation:** Add an `NSMenu` for right-click with: Settings (Cmd+,), Quit (Cmd+Q).

### 1.4 Proper app menu bar
**[Interaction]**

When the Settings window is focused, the app should present a standard macOS menu bar with App menu (About, Settings, Quit), Edit (for text fields), Window, and Help menus.

**Recommendation:** Create `NSMenu` for the main menu bar that appears when the settings window is key. This is required by HIG.

## Priority 2 — Native polish (makes the app feel professional)

### 2.1 Grouped form style for Settings
**[Interaction, Visuals]**

The Settings window should use `.formStyle(.grouped)` with `Section` headers for a native macOS settings appearance. The current SwiftUI layout uses manual `GroupBox` which looks non-standard.

**Recommendation:** Restructure SettingsView to use:
```swift
Form {
    Section("Exported Folders") { ... }
    Section("NFS Network Access") { ... }
}
.formStyle(.grouped)
```

### 2.2 Keyboard shortcuts throughout
**[Interaction, Inclusivity]**

Every action should be keyboard-accessible. The Settings window should support Tab navigation between fields and Enter to save.

**Recommendation:** Add `.keyboardShortcut()` to all buttons. Ensure Tab order is logical. Add Cmd+N for "Add Folder" in settings.

### 2.3 VoiceOver accessibility
**[Inclusivity]**

All UI elements need accessibility labels and hints. The menu bar panel needs logical grouping for VoiceOver's container-based navigation.

**Recommendation:**
- Add `accessibilityLabel` to the status bar icon ("iCloud NFS Exporter — daemon running/stopped").
- Add `accessibilityElement(children: .contain)` to logical groups in the panel.
- Add `accessibilityHint` to action buttons.
- Test with VoiceOver enabled (Cmd+F5).

### 2.4 System colours and Dark Mode
**[Visuals, Inclusivity]**

Verify that all UI uses semantic colours (`.primary`, `.secondary`) rather than hardcoded values. The settings window should look correct in both light and dark mode.

**Recommendation:** Audit all `Color` usage. Replace any hardcoded colours with system semantic colours. Test in both appearances via System Settings > Appearance.

### 2.5 Reduced Motion support
**[Inclusivity]**

If any animations are added, they must respect the Reduce Motion accessibility setting.

**Recommendation:** Check `@Environment(\.accessibilityReduceMotion)` before animating. Fall back to crossfades.

## Priority 3 — Delight (elevates from "tool" to "experience")

### 3.1 Live status indicator in menu bar
**[Delight, Visuals]**

Instead of a static cloud icon, show live status:
- Hollow cloud outline: daemon not running
- Solid cloud: daemon running, idle
- Cloud with animated arrow: file currently hydrating
- Cloud with exclamation mark: error state

**Recommendation:** Create 4 SF Symbol variants using `cloud`, `cloud.fill`, `icloud.and.arrow.down`, `exclamationmark.icloud`. Update the icon reactively based on daemon state and active hydrations.

### 3.2 Notifications for hydration events
**[Interaction, Delight]**

When a file finishes hydrating (especially large files), show a macOS notification: "Report.pdf is now available via NFS".

**Recommendation:** Use `UNUserNotificationCenter` with actionable notifications. Allow users to disable notifications in Settings.

### 3.3 Spotlight integration
**[Innovation, Interaction]**

Register exported folders and their hydration status with Core Spotlight so users can search "iCloud NFS" in Spotlight and see status.

**Recommendation:** Use `CSSearchableItem` to index exported folders. Update the index when status changes.

### 3.4 Drag-and-drop folder addition
**[Interaction]**

In the Settings window, allow dragging a Finder folder onto the folder list to add it as an export.

**Recommendation:** Add `.onDrop(of: [.fileURL])` to the folder list view. Validate that the dropped path is within iCloud Drive.

### 3.5 Onboarding experience
**[Delight, Interaction]**

On first launch (no config), show a focused onboarding flow rather than jumping straight to a settings form. Walk through:
1. Welcome — what the app does (one sentence + illustration)
2. Select folders — visual folder picker with previews
3. Network — CIDR input with explanation for non-technical users
4. Done — summary with "Start Exporting" button

**Recommendation:** Create an `OnboardingView` with `TabView` pages. Show on first launch, then never again. Settings window remains for subsequent changes.

## Priority 4 — Future vision

### 4.1 Liquid Glass adoption (macOS Tahoe)
When targeting macOS 26, adopt the Liquid Glass design language for the menu bar panel and settings window. This will make the app feel current and integrated with the latest OS design.

### 4.2 Multi-platform expansion
Consider an iOS/iPadOS companion app that shows export status and allows configuration. ADA winners consistently demonstrate thoughtful multi-platform presence.

### 4.3 Shortcuts.app integration
Expose "Add Folder", "Start Export", "Check Status" as Shortcuts actions via `AppIntents`. This enables automation and Siri integration.

### 4.4 WidgetKit
A macOS Desktop Widget showing current export status and hydration activity.

---

## Implementation Order

For v0.2.0, focus on Priority 1 (must-fix) and Priority 2 (native polish). These changes transform the app from "functional CLI wrapper" to "feels like it belongs on macOS."

Priority 3 items are individually small but collectively create the emotional resonance that ADA judges look for. Tackle them in v0.3.0.

Priority 4 items are forward-looking and depend on macOS release cycles.

**Estimated scope for v0.2.0:** 1.1 (rich menu panel), 1.2 (template icon), 1.3 (right-click), 1.4 (app menu), 2.1 (grouped form), 2.2 (keyboard shortcuts), 2.3 (VoiceOver), 2.4 (dark mode audit).
