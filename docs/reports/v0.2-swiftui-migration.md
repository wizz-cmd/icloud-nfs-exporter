# v0.2 Implementation Report — SwiftUI @main Migration

**Version:** 0.2.0
**Date completed:** 2026-04-16
**Commit:** `0c08a94` on `main`
**CI run:** Passed (54s on macOS 15) — 62 tests (17 Swift + 21 Rust + 24 Python)

## Objective

Replace the manual AppKit lifecycle (`main.swift` + `NSApplicationDelegate`) with a modern SwiftUI `@main App` struct, gaining native macOS features — standard menu bar, Settings scene, rich menu bar panel, `@Observable` state, and VoiceOver accessibility — that the AppKit approach could not provide without significant manual effort.

## Motivation

The v0.1.x app had five design problems identified against the [Apple Human Interface Guidelines](../research/macos-app-design-rules.md) and [Apple Design Award criteria](../research/apple-design-award-criteria.md):

1. **No standard app menu bar** — missing About, Edit, Window, Help menus (HIG rule 1.6)
2. **No native Settings scene** — Cmd+, didn't work; settings opened via manual NSWindow (HIG rule 1.7)
3. **Plain NSMenu for status** — disabled text items, no visual richness (ADA: Interaction)
4. **Zero accessibility** — no VoiceOver labels, no keyboard navigation (HIG rule 5.1)
5. **Manual state management** — instance variables on NSApplicationDelegate, no reactive updates

## Architecture Change

### Before (v0.1.x)

```
main.swift                          → NSApplication.shared + AppDelegate
AppDelegate.swift                   → NSStatusItem + NSMenu + NSWindow
SettingsView.swift (SwiftUI)        → hosted in NSHostingView inside NSWindow
ConfigReader.swift                  → TOML parser
```

Entry point: imperative AppKit `app.run()`. State scattered across `AppDelegate` instance variables. Settings opened by manually creating and centering an `NSWindow`. Menu rebuilt by constructing `NSMenuItem` objects. No standard macOS menu bar.

### After (v0.2.0)

```
MenuBarApp.swift (@main App)        → MenuBarExtra + Settings scenes
AppState.swift (@Observable)        → centralised reactive state
StatusPanelView.swift (SwiftUI)     → rich menu bar panel
SettingsView.swift (SwiftUI)        → TabView container with Save/Revert
GeneralSettingsTab.swift            → socket path, mount base
FoldersSettingsTab.swift            → folder list + iCloud discovery
NetworkSettingsTab.swift            → NFS CIDR config
AdvancedSettingsTab.swift           → daemon status + refresh
ConfigReader.swift                  → unchanged (+ Equatable on FolderEntry)
```

Entry point: declarative `@main struct MenuBarApp: App`. Single `@Observable AppState` drives all views reactively. `MenuBarExtra(.window)` provides a rich SwiftUI dropdown panel. `Settings` scene gives native Cmd+, handling and standard window behaviour. SwiftUI automatically provides the standard app menu bar (About, Edit, Window, Help).

## Components

### MenuBarApp.swift (entry point)

SwiftUI `@main App` with two scenes:

- **`MenuBarExtra`** with `.menuBarExtraStyle(.window)` — renders `StatusPanelView` as a popover-like panel when the menu bar icon is clicked. The `.window` style enables full SwiftUI layout (unlike `.menu` which is constrained to `NSMenu` semantics).
- **`Settings`** — renders `SettingsView`. Automatically handles Cmd+, shortcut, window lifecycle, and inclusion in the Window menu.

`AppState` is held as `@State` on the App struct and injected into both scenes via `.environment()` (the `@Observable` environment path).

### AppState.swift (state management)

`@Observable` class replacing all AppDelegate state:

| Property | Type | Purpose |
|---|---|---|
| `config` | `ConfigReader` | Loaded TOML configuration |
| `daemonRunning` | `Bool` | Polled every 10s via `IPCClient.isAvailable` |
| `needsOnboarding` | `Bool` | True when no config file exists on launch |
| `statusIcon` | `String` (computed) | SF Symbol name: `cloud` or `cloud.fill` |
| `statusDescription` | `String` (computed) | VoiceOver label for the status icon |

| Method | Purpose |
|---|---|
| `refresh()` | Reload config from disk + re-poll daemon |
| `saveConfig(_:)` | Write updated config to disk |
| `startPolling()` | Begin 10-second timer for daemon health checks |
| `checkStatus()` | Async IPC ping on utility queue, updates `daemonRunning` on main thread |

The `@Observable` macro (macOS 14+) makes all property mutations automatically trigger SwiftUI view updates — no manual `objectWillChange.send()` or `@Published` needed.

### StatusPanelView.swift (menu bar panel)

SwiftUI view rendered inside `MenuBarExtra(.window)`:

- **Status header** — SF Symbol icon with hierarchical rendering + "Running"/"Stopped" text
- **Folder list** — label + mount point for each configured folder
- **Actions** — Refresh (Cmd+R), Settings (Cmd+,) via `SettingsLink`, Quit (Cmd+Q)
- **First-launch** — `.onAppear` calls `openSettings()` if `needsOnboarding` is true

Width: 280pt. All elements have VoiceOver accessibility labels and hints.

### SettingsView.swift (settings container)

`TabView` with four tabs, matching macOS Settings conventions (HIG rule 1.7):

| Tab | SF Symbol | Content |
|---|---|---|
| General | `gear` | Socket path, mount base directory |
| Folders | `folder` | Folder list with add/remove, iCloud container discovery |
| Network | `network` | NFS allowed network CIDR |
| Advanced | `gearshape.2` | Daemon status indicator, refresh button |

Each tab uses `Form { Section(...) { } }.formStyle(.grouped)` for the standard macOS grouped form appearance.

Settings uses a deferred save model: edits are held in local `@State` until the user clicks Save. A Revert button reloads from `AppState.config`. The `hasChanges` flag enables/disables both buttons.

### Settings Tabs

**GeneralSettingsTab** — two text fields (socket path, mount base) with accessibility labels and hints.

**FoldersSettingsTab** — `ForEach` over configured folders with `LabeledContent` showing label, mount point, and a remove button. An "Add iCloud Folder" `Menu` discovers available containers via `ConfigReader.discoverContainers()` and filters out already-added folders.

**NetworkSettingsTab** — CIDR text field with explanatory caption.

**AdvancedSettingsTab** — reads `AppState` directly from `@Environment` to show live daemon status (green/red dot + text) and socket path. Refresh button triggers `appState.refresh()`.

## Deployment Target

Bumped from macOS 13 (Ventura) to macOS 14 (Sonoma) across all three Swift packages:

- `src/app/Package.swift`
- `src/hydration/Package.swift`
- `src/helper/Package.swift`

**Justification:** macOS 14 Sonoma was released October 2023 (2.5 years ago). The bump unlocks `@Observable`, `SettingsLink`, and improved SwiftUI APIs. No HydrationCore source changes were required — the library compiles identically on both targets.

The `LSMinimumSystemVersion` in `scripts/create-app-bundle.sh` was also updated to `14.0`.

## Accessibility

Every interactive element now has VoiceOver support:

| Element | Accessibility |
|---|---|
| Menu bar icon | `accessibilityLabel` with current daemon state |
| Status header | `accessibilityElement(children: .combine)` merging icon + text |
| Folder rows | Combined label: "folder name, mounted at path" |
| Remove buttons | `accessibilityLabel("Remove FolderName")` + hint |
| Add folder menu | `accessibilityLabel("Add iCloud folder to export")` |
| Network field | Label + hint explaining CIDR notation |
| Daemon status | Combined element with running/stopped state |
| All action buttons | `keyboardShortcut` for keyboard navigation |
